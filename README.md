# ATMOS — приложение учёта продаж, консультаций и операций

Клиент-серверное веб-приложение для клиники пластической хирургии ATMOS: единая
база пациентов, консультаций, операций и платежей с разграничением прав, аудитом,
мягким удалением, автоматическими расчётами и ежедневным дашбордом для администратора.

## Технологии

- **Клиент:** React 18 + TypeScript + Vite, Tailwind CSS, TanStack Query, Recharts
- **Сервер:** Node.js 20 + Express + TypeScript, Prisma ORM, JWT + bcrypt (RBAC)
- **БД:** PostgreSQL 16
- **Запуск:** Docker Compose (db, server, client)

## Быстрый запуск

Требуется только **Docker Desktop** (Node.js локально не нужен).

```bash
docker compose up --build
```

При первом старте автоматически: применяется схема БД (`prisma db push`),
загружаются справочники и демо-данные (сид).

Откройте:

- Клиент: <http://localhost:5173>
- API (проверка): <http://localhost:4000/api/health>

### Демо-доступы

| Роль | Логин | Пароль |
|---|---|---|
| Администратор | `admin` | `admin123` |
| Оператор | `operator` | `operator123` |

> После первого входа смените пароли (раздел «Пользователи и справочники»).

## Роли (RBAC, проверка на сервере)

- **Оператор** — вводит данные; редактирует только **свою** запись **в день её
  создания**; удаление недоступно.
- **Администратор** — полный доступ: правка любой записи, мягкое удаление и
  восстановление, справочники, пользователи, аудит.

## Экраны

Дашборд · Пациенты (+ карточка) · Консультации · Операции · Предоплаты и остатки ·
Касса · Проверка ошибок · Аудит (админ) · Пользователи и справочники (админ).

## Вычисляемые поля (не редактируются вручную)

`к оплате = стоимость + наркоз` · `оплачено = Σ платежей` ·
`остаток = к оплате − оплачено` · `оплачено 100% при остатке ≤ 0` ·
`месяц` из даты операции · конверсия и выручка на дашборде.


## Примечание по окружению (Windows / OneDrive)

Проект лежит в папке OneDrive с кириллицей и пробелами. Bind-mount такой папки в
Docker на Windows **нестабилен**: через время (ре-синхронизация OneDrive)
монтирование слетает и контейнер теряет файлы. Поэтому по умолчанию исходники
**«запечены» в образы** (`COPY . .` в Dockerfile), а `docker-compose.yml` их не
монтирует — приложение не зависит от OneDrive и запускается стабильно.

Следствие: правки кода применяются после пересборки образов:

```bash
docker compose up -d --build
```

### Разработка с hot reload

Нужен мгновенный отклик на правки? Перенесите проект в путь без OneDrive/кириллицы
(например `C:\atmos-uchet`) и запускайте с dev-override (bind-mount + hot reload):

```bash
docker compose -f docker-compose.yml -f docker-compose.dev.yml up
```
