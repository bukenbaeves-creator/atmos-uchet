# Деплой на Render

Приложение развёрнуто как **один web-сервис** (Express отдаёт и API, и собранный
фронтенд) + управляемый **PostgreSQL**. Конфигурация — в `render.yaml` и `Dockerfile.prod`.

## Шаги

1. Зарегистрируйтесь на <https://render.com> и подключите свой GitHub.
2. **New → Blueprint** → выберите репозиторий `atmos-uchet`. Render прочитает
   `render.yaml` и создаст:
   - базу `atmos-db` (PostgreSQL);
   - web-сервис `atmos` (сборка из `Dockerfile.prod`).
3. Нажмите **Apply**. Первый деплой соберёт образ (5–10 мин): применит схему БД,
   засеет справочники и учётку администратора (демо-данные в проде НЕ создаются).
4. Откройте выданный адрес `https://atmos-XXXX.onrender.com` → войдите
   `admin / admin123`.

## Сразу после первого входа

- Смените пароль администратора (Пользователи и справочники → Пользователи).
- Смените **коды регистрации** (там же, блок «Коды регистрации»).
- `JWT_SECRET` Render генерирует автоматически — менять не нужно.

## Важно

- **План `free`**: web-сервис засыпает при простое (первый запрос ~30 сек),
  а бесплатная БД временная. Для клиники выберите платный план БД и сервиса
  (в `render.yaml` поле `plan`).
- **Данные пациентов (ПДн)** окажутся на стороннем облаке — выбирайте регион и
  провайдера с учётом требований к персональным данным. Для строгого хранения
  «в клинике» используйте локальный сервер (`docker compose up` из
  `docker-compose.yml`).
- Резервные копии БД настройте на стороне Render (или регулярный `pg_dump`).

## Локальный запуск (для сравнения)

```bash
docker compose up --build           # dev: клиент :5173, API :4000, Postgres
```
